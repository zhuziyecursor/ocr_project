### 2026-04-15 项目初始化与前端框架搭建

**背景：** 搭建 Vue 3 + Vite + Element Plus + TailwindCSS 前端框架

**遇到的问题：**
1. `main.js` 由 AI 生成时只写了 import，缺少 `createApp` + `mount`，导致页面空白
2. TailwindCSS v4 的 Vite 插件方式在 dev server 下不稳定，改用 PostCSS 方式解决

**决策/结论：** 
- main.js 必须包含 `createApp` → `app.use(...)` → `app.mount('#app')` 完整链路
- TailwindCSS v4 统一使用 `@tailwindcss/postcss` + `postcss.config.js`，不用 Vite 插件

**注意事项：** 见 `doc/pitfalls.md` 问题一、二

### 2026-04-15 后端 Celery Worker 首次联调

**背景：** 上传接口首次端到端联调

**遇到的问题：**
1. Celery Worker 未启动 → 上传接口 500
2. `wrapper.py` 调用 OpenDataLoader 参数名错误（`pdf_path` 应为 `input_path`）

**决策/结论：**
- 确立"所有第三方库封装必须有运行时验证"原则，不能只看代码逻辑
- 明确开发启动依赖：需要同时启动 OCR 引擎 + 后端 + Celery Worker + 前端

**注意事项：** 见 `doc/pitfalls.md` 问题四

---

### 2026-04-16 PDF 渲染失败 — Vue Proxy 与 pdfjs 私有字段冲突

**背景：** 前端核验页左侧 PDF 渲染失败

**遇到的问题：**
- `ref()` 对 pdfjs 对象做深度 Proxy，导致私有字段 `#pagePromises` 无法访问
- 同事排查一上午，方向集中在 pdfjs 版本/Vite 配置，均无效

**决策/结论：**
- 凡存入 Vue 响应式的第三方库对象（含私有字段/原生 API），统一用 `shallowRef()` 或 `markRaw()`
- `shallowRef` 包裹的数组，push 不触发更新，需重新赋值

**注意事项：** 见 `doc/重难点问题排查思路.md` 问题一

## AI Coding 方法论心得

> 持续补充，形成个人知识库

### 心得一：边界驱动排查

遇到 bug 先找"能跑/不能跑"的最小差异边界，范围立即缩小一半。本项目 PDF 渲染问题就是靠这个方法快速定位到 Vue 层。

### 心得二：相信报错信息

报错说什么就是什么，不要绕开它。"class did not declare it" = `this` 被替换 = 找谁包装了对象。不要在版本、配置上绕圈子。

### 心得三：文档与代码必须同步

代码改了，相关文档（启动方式、参数名、接口格式）必须同步更新。文档和代码出入时，先确认哪个是真实状态，再改另一个。

### 心得四：AI 生成代码必须验证

AI 生成的代码可能：只有 import 没有执行、参数名拼错、逻辑看起来对但运行时失败。每次生成后必须运行时验证，不能只看静态代码。

### 心得五：围绕 MVP 手册开发

遇到需求分歧时，回到 `MVP手册.md` 和 `技术方案手册.md`。是需求变了就改手册，是实现偏了就改代码，不要默默绕过。
