## 1. MVP 定义

### 1.1 MVP 核心原则

MVP 聚焦于**最小可验证的核心流程**：上传文件 → 自动识别 → 分屏比对 → 人工核验 → 导出。去除所有可延后的功能，快速验证产品价值。

### 1.2 MVP 功能范围

#### ✅ MVP 包含

| 功能 | 说明 | 优先级 |
|------|------|-------|
| 文件上传 | PDF/图片上传，同步+异步处理 | P0 |
| 文档识别 | OpenDataLoader 识别，文本/表格/图片 | P0 |
| 单文件比对 | 原文 vs 识别结果，分屏界面 | P0 |
| 高亮差异 | 差异内容自动高亮，颜色区分类型 | P0 |
| 点击联动 | 左右分屏互相定位 | P0 |
| 人工修正 | 直接编辑识别结果 | P0 |
| 修改历史 | 记录每次修改，支持查看 | P0 |
| 任务进度 | WebSocket 实时推送处理进度 | P0 |
| Markdown 导出 | 导出含核验记录的 Markdown | P0 |
| 置信度标记 | 低置信度内容自动标记需核验 | P1 |
| 跨文件比对 | 文件A vs 文件B | P1 |
| 任务列表 | 查看历史上传记录 | P1 |

#### ❌ MVP 不包含

| 功能 | 延后理由 |
|------|---------|
| ~~多角色权限管理~~ | **确认不需要** |
| 跨页表格合并 | 实现复杂，MVP先绕过 |
| 页面对齐映射模板保存 | 非核心功能 |
| ~~批量上传~~ | **后续迭代实现** |
| ~~邮件/企业微信通知~~ | **确认不需要** |
| ~~SSO 对接~~ | **暂不对接** |
| 印章真伪鉴定 | 明确告知客户不做（只提取文字+need_review） |

## 4. MVP 功能详细说明

### 4.1 文件上传

**输入**：用户选择文件  
**处理**：
1. 检测文件大小和页数
2. ≤5MB/≤10页 → 同步处理，等待结果
3. 其余 → 异步入队，返回任务ID

**输出**：任务ID，重定向到核验界面（同步）或任务列表（异步）

**错误处理**：
- 不支持的格式：提示并拒绝
- 超过大小限制（500MB）：提示并拒绝
- 上传失败：提示重试

### 4.2 文档识别（MVP 简化版）

**处理流程**：
```python
def process_document(file_path):
    # Step 1: OpenDataLoader 识别
    raw_result = opendataloader_pdf.convert(
        file_path,
        format="json",
        image_output="embedded",
        table_method="cluster"
    )
    
    # Step 2: 审计增强（MVP简化版）
    enhanced = basic_audit_enhance(raw_result)
    
    # Step 3: 存入数据库
    save_to_db(enhanced)
```

**MVP 简化的置信度规则（3条核心规则）**：
1. 来自 OCR 层（扫描件）→ 置信度降至 0.88
2. 内容包含手写特征（字体非标准）→ 强制 `need_review: true`
3. 印章类型 → 强制 `need_review: true`

### 4.3 分屏比对界面（MVP 核心功能）

**左侧 PDF 渲染**：
- 使用 PDF.js 渲染原始 PDF
- Canvas 层 + SVG overlay 显示高亮框
- 支持缩放（+/-按钮，滚轮缩放）
- 差异区域用彩色框标注

**右侧结构化内容**：
- 文本块：`<div>` 渲染，支持直接编辑
- 表格块：`<table>` 渲染，单元格可编辑
- 图片块：按 bbox 尺寸渲染原始图片
- 图表块：占位符 + AI 描述文字

**联动规则**：
- 点击左侧任意区域 → 右侧滚动到对应块并高亮
- 点击右侧任意块 → 左侧高亮对应区域
- 滚动任意一侧 → 另一侧基于锚点同步

**差异高亮颜色**：
- 🟡 黄色：内容修改（最常见）
- 🟢 绿色：新增内容
- 🔴 红色：删除内容
- 🔵 蓝色：需人工核验

### 4.4 人工核验

**核验操作**：

| 操作 | 触发方式 | 效果 |
|------|---------|------|
| 确认正确 | 点击 [✓] | 标记已核验，高亮消除 |
| 修改内容 | 双击内容 | 进入编辑模式 |
| 保存修改 | 点击 [保存] / Enter | 保存并记录历史 |
| 取消编辑 | 按 Escape | 恢复原内容 |

**修改历史**（每条记录）：
```
时间 | 修改人 | 原内容 | 新内容 | 所在页 | block_id
```

**导出条件**：
- 默认：所有 `need_review: true` 已确认
- 提供强制导出按钮（带警告提示）

### 4.5 进度推送

**WebSocket 消息格式**：
```json
ƒ
  "status": "PROCESSING",
  "current_page": 15,
  "total_pages": 45,
  "message": "正在识别第15页..."
}
```

**前端处理**：
```javascript
const ws = new WebSocket(`ws://api/tasks/${taskId}/progress`)

ws.onmessage = (event) => {
  const data = JSON.parse(event.data)
  
  if (data.status === 'DONE') {
    // 跳转核验界面
    router.push(`/review/${data.document_id}`)
  } else if (data.status === 'FAILED') {
    // 显示错误，提供重试
    showError(data.message)
  } else {
    // 更新进度条
    updateProgress(data.current_page / data.total_pages)
  }
}
```

## 5. MVP 数据模型

### 5.1 核心数据表

**documents**
```sql
CREATE TABLE documents (
    id UUID PRIMARY KEY,
    file_name VARCHAR(500) NOT NULL,
    file_path VARCHAR(1000) NOT NULL,   -- MinIO路径
    file_size BIGINT,
    total_pages INT,
    status VARCHAR(20),                 -- PENDING/PROCESSING/DONE/FAILED
    uploaded_by VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

**recognition_results**
```sql
CREATE TABLE recognition_results (
    id UUID PRIMARY KEY,
    document_id UUID REFERENCES documents(id),
    result_json JSONB NOT NULL,         -- OpenDataLoader输出 + 审计增强字段
    created_at TIMESTAMP DEFAULT NOW()
);
```

**review_records**
```sql
CREATE TABLE review_records (
    id UUID PRIMARY KEY,
    document_id UUID REFERENCES documents(id),
    block_id VARCHAR(100),
    page_no INT,
    original_content TEXT,
    modified_content TEXT,
    reviewed_by VARCHAR(100),
    reviewed_at TIMESTAMP DEFAULT NOW()
);
```

### 5.2 识别结果扩展字段

OpenDataLoader 原始输出基础上，增加审计字段：

## 8. MVP 验收标准

### 8.1 功能验收

| 验收项 | 标准 |
|--------|------|
| 文件上传 | 支持PDF/图片，≤10页文件30秒内完成 |
| 文本识别 | 标准印刷文字准确率 ≥ 99% |
| 表格识别 | 有线框表格准确率 ≥ 92% |
| 分屏界面 | 左右联动响应 < 200ms |
| 同步滚动 | 滚动时两侧对应内容保持对齐 |
| 修正功能 | 编辑保存后立即生效，历史记录可查 |
| Markdown导出 | 导出内容完整，包含核验记录 |

### 8.2 性能验收

| 指标 | 标准 |
|------|------|
| 小文件处理 | ≤30秒（≤10页PDF） |
| 大文件处理 | 100页PDF ≤10分钟 |
| 并发用户 | 10人同时操作无明显卡顿（MVP降低要求） |
| 界面响应 | 首屏加载 ≤3秒 |

### 8.3 MVP 不验收项

- 100人并发（MVP阶段降低要求）
- 跨文件比对准确率
- 手写文字识别准确率

## 9. MVP 已知限制

在 MVP 演示前，需向客户说明以下限制：

1. **图表内容**：柱状图/折线图右侧仅显示AI描述文字，无法还原图表
2. **手写文字**：识别准确率未经系统性验证，强制标记需人工核验
3. **印章真伪**：系统仅提取印章文字，不做真伪鉴定
4. **跨文件比对**：MVP 版本功能较基础，页面对齐准确率依赖大模型质量
5. **并发上限**：MVP 阶段支持约10人并发，生产版本扩展到100人
