# 审计文档处理系统 - 需求分析手册

> 版本：v1.0  
> 创建日期：2026-04-14  
> 基于：PRD v0.1-draft + OpenDataLoader PDF 技术调研

---

## 1. 项目概述

### 1.1 业务背景

审计行业日常需处理大量文档（财务报表、审计底稿、票据凭证、合同等），目前主要依赖人工逐页核对，存在以下痛点：

- **效率低下**：人工核对速度慢，500人团队日处理20,000页文档压力大
- **易出错**：人为疏漏导致关键差异遗漏
- **无法追溯**：缺乏修改历史和审计日志
- **重复劳动**：相似文档需重复核验

### 1.2 核心目标

| 目标 | 量化指标 |
|------|---------|
| 自动化识别 | 标准印刷文字准确率 ≥ 99%，表格识别准确率 ≥ 92% |
| 智能比对 | 支持页级/段落级/表格单元格级差异比对 |
| 人工核验 | 提供分屏界面，左右联动，点击定位 |
| 结构化输出 | 导出标准 Markdown 格式，对接下游系统 |
| 处理性能 | 小文件(≤10页) ≤30秒，日处理能力 ≥20,000页 |

### 1.3 系统边界

**本期范围**：
- 文件上传（PDF/Word/Excel/图片）
- 自动识别（文本/表格/图片）
- 单文件比对（原文 vs 识别结果）
- 跨文件比对（文件A vs 文件B）
- 人工核验与修正
- Markdown 导出

**本期不包含**：
- 审计报告自动生成
- 审批流程
- 对外 API 开放平台
- 移动端支持

## 2. 用户角色与场景

### 2.1 核心用户角色

| 角色 | 职责 | 核心操作 |
|------|------|---------|
| 核验员 | 审计人员，负责文档核验 | 上传文件、查看比对结果、修正错误、导出报告 |

**待确认**：是否需要管理员角色（用户管理、规则配置）

### 2.2 典型使用流程

```
核验员上传文件
    ↓
系统自动识别（文本/表格/图片）
    ↓
生成比对结果（差异高亮）
    ↓
分屏界面核验（左侧原文，右侧识别结果）
    ↓
点击差异处修改
    ↓
导出 Markdown 文件
```

### 2.3 关键使用场景

#### 场景1：单文件核验
**目标**：验证 OCR 识别准确性  
**流程**：上传扫描件 → 系统识别 → 左右对比 → 修正错误 → 导出

#### 场景2：跨文件比对
**目标**：验证两份文档数据一致性（如审计底稿 vs 财务报表）  
**流程**：上传两份文件 → 页面对齐 → 元素匹配 → 差异标记 → 逐项核验

#### 场景3：大文件异步处理
**目标**：处理500+页审计底稿  
**流程**：上传文件 → 进入任务队列 → WebSocket 推送进度 → 处理完成通知 → 开始核验

## 3. 功能需求详解

### 3.1 文件上传与管理

#### 3.1.1 支持格式

| 类型 | 格式 | 处理方式 |
|------|------|---------|
| PDF | .pdf | 原生PDF直接提取，扫描PDF走OCR |
| Office | .docx, .xlsx | 结构化解析 |
| 图片 | .jpg, .png, .tiff | OCR识别 |

#### 3.1.2 处理策略

| 条件 | 处理方式 | 用户体验 |
|------|---------|---------|
| ≤5MB 且 ≤10页 | 同步处理 | 等待30秒内，直接跳转比对界面 |
| >5MB 或 >10页 | 异步处理 | 进入任务列表，WebSocket推送进度 |

#### 3.1.3 任务状态

```
PENDING → PROCESSING → DONE
                    → FAILED (支持重试)
```

### 3.2 文档解析与识别

#### 3.2.1 技术选型：OpenDataLoader PDF

**选择理由**：
- 开箱即用，无需自研底层引擎
- 表格识别准确率 92.8%（TEDS score）
- 原生支持坐标输出（bbox）
- Hybrid 模式自动分流（简单页面本地处理，复杂页面AI后端）
- 支持 base64 图片嵌入

**核心能力**：
```python
result = opendataloader_pdf.convert(
    pdf_path,
    format="json",
    image_output="embedded",  # base64图片
    table_method="cluster",   # 复杂表格识别
    hybrid="docling-fast"     # 启用AI增强
)
```

#### 3.2.2 识别内容类型

| 内容类型 | 识别能力 | 输出格式 |
|---------|---------|---------|
| 文本 | 标准印刷文字 | content + bbox |
| 表格 | 有线框/无线框/合并单元格 | rows/cells + row_span/col_span |
| 图片 | 提取原始图片 | base64 data + bbox |
| 图表 | AI生成描述（无法提取原始图表） | description + bbox |
| 印章 | 文字提取（需人工确认真伪） | content + need_review标记 |

### 3.3 比对核验

#### 3.3.1 比对模式

| 模式 | 说明 | 技术实现 |
|------|------|---------|
| 单文件比对 | 原文 vs 识别结果 | 基于 block_id 直接映射 |
| 跨文件比对 | 文件A vs 文件B | 语义匹配 + 本地大模型辅助 |

#### 3.3.2 比对粒度

| 粒度 | 适用场景 | 算法 |
|------|---------|------|
| 页级 | 快速定位差异页 | 页面内容哈希对比 |
| 段落级 | 文本差异 | Myers diff 算法 |
| 表格全量 | 财务数据核对 | 单元格逐一比对 + 数字标准化 |

#### 3.3.3 跨文件比对流程

```
1. 页面对齐
   ├─ 本地大模型分析各页内容
   ├─ 自动推荐页面映射关系
   └─ 用户手动确认/调整

2. 元素语义匹配
   ├─ 计算文本相似度（编辑距离）
   ├─ 计算位置相似度（bbox相对位置）
   └─ 加权得分 > 0.75 视为匹配

3. 差异计算
   ├─ 文本：Myers diff
   ├─ 表格：数字标准化后逐单元格比对
   └─ 图片：仅比对描述文字
```

#### 3.4.2 交互功能

| 操作 | 说明 |
|------|------|
| 点击左侧 | 右侧滚动到对应位置并高亮 |
| 点击右侧 | 左侧高亮对应区域 |

### 3.5 结果导出

#### 3.5.1 导出格式

**格式**：Markdown (.md)

**结构**：
```markdown
# 文档识别结果

## 文档信息
- 原始文件：财务报表2024.pdf
- 处理时间：2026-04-14 10:30:00
- 总页数：12
- 核验状态：已完成（23处差异，23处已处理）

## 第1页

### 文本内容
货币资金：1,234,567
应收账款：987,654

### 表格内容
| 项目 | 年初余额 | 年末余额 |
|------|---------|---------|
| 货币资金 | 1,200,000 | 1,234,567 |

### 图片内容
- [印章] XX会计师事务所（需人工确认）

## 核验记录
| 页码 | 位置 | 原识别 | 修正内容 | 核验人 | 时间 |
|------|------|--------|---------|-------|------|
| 1 | 第3行 | 1,284,567 | 1,234,567 | 张三 | 2026-04-14 11:00 |
```

#### 3.5.2 导出条件

- 默认：所有 `need_review: true` 的内容已被人工确认
- 支持强制导出（标注"含未核验内容"）

## 4. 非功能性需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 同步处理响应时间 | ≤ 30秒（≤10页） |
| 异步处理吞吐量 | ≥ 20,000页/天 |
| 分屏界面加载 | ≤ 3秒（首屏） |
| 高峰并发用户 | 100人同时操作 |
| WebSocket延迟 | ≤ 1秒 |

### 4.2 可靠性需求

| 指标 | 要求 |
|------|------|
| 系统可用性 | ≥ 99.5% |
| 任务失败重试 | 最多3次 |
| 数据持久化 | 原始文件及识别结果永久保存 |

### 4.3 安全需求

- 数据不出内网（私有化部署）
- 所有操作记录审计日志
- 用户只能访问自己上传的文件

### 4.4 兼容性需求

- 前端：Chrome/Edge 最新版
- 部署：Linux + Docker
- 本地大模型：OpenAI 兼容 API

## 5. 关键约束与风险

### 5.1 技术约束

| 约束 | 影响 | 缓解方案 |
|------|------|---------|
| OpenDataLoader 不输出置信度 | 无法自动标记低质量识别 | 自研启发式置信度计算算法 |
| 图表无法还原 | 右侧只能显示描述文字 | 占位符 + 提示查阅左侧原文 |
| 手写识别准确率未知 | 可能达不到82-88%预期 | POC阶段验证，不达标则集成专项模型 |

### 5.2 业务风险

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 客户对图表无法还原不接受 | 中 | 高 | POC阶段明确说明，提供左侧原文查阅 |
| 跨文件比对页面对齐不准确 | 中 | 中 | 提供手动调整界面，保存常用映射模板 |
| 大文件处理超时 | 低 | 中 | 分页处理策略，每次50页 |
