## 一、项目概述

### 1.1 项目背景
本系统旨在为审计场景提供高精度的文档识别与智能比对能力,解决传统审计工作中文档处理效率低、人工比对易出错的痛点。

### 1.2 核心目标
- 实现原生PDF与扫描PDF的自动识别与高精度OCR
- 支持复杂表格、图片、印章等多元素识别
- 提供分屏可视化比对界面,支持坐标级联动
- 实现跨文件智能比对与差异标注
- 支持私有化部署,确保数据安全

### 1.3 技术选型决策

**核心引擎**: OpenDataLoader (替代原PaddleOCR方案)

**选型理由**:
- 内置原生PDF/扫描PDF自动识别,无需手动路由
- Hybrid模式自动分流:简单页面本地处理(0.02s/页),复杂页面AI后端
- 原生支持带坐标的JSON输出,每个元素都有bbox
- 表格识别准确率更高(92.8% TEDS score)
- 可节省50%+底层开发工作量

**技术栈**:
- 后端: FastAPI + Celery + Redis + PostgreSQL
- 前端: Vue 3 + Vite + Pinia + PDF.js + TailwindCSS
- OCR引擎: OpenDataLoader (hybrid mode)
- 大模型: 客户本地大模型(OpenAI兼容API)
- 部署: Docker Compose

```python
# audit_wrapper.py
class AuditEnhancer:
    def enhance(self, opendataloader_output):
        """给OpenDataLoader输出添加审计字段"""
        
        # 1. 计算置信度(基于启发式规则)
        for block in blocks:
            block['confidence'] = self._compute_confidence(block)
            block['need_review'] = block['confidence'] < 0.85
        
        # 2. 添加审计元数据
        result['audit_metadata'] = {
            'processing_mode': 'hybrid',
            'extraction_timestamp': datetime.now(),
            'pdf_version': self._get_pdf_version(),
            'total_pages': len(pages)
        }
        
        # 3. 质量检测
        result['quality_flags'] = self._detect_quality_issues()
        
        return result
    
    def _compute_confidence(self, block):
        """多维度置信度计算"""
        score = 1.0
        
        # 维度1: 是否来自OCR(扫描件)
        if block.get('hidden_text'):
            score *= 0.88
        
        # 维度2: 字体一致性
        if self._is_rare_font(block['font']):
            score *= 0.90
        
        # 维度3: bbox异常检测
        if self._has_bbox_anomaly(block['bbox']):
            score *= 0.80
        
        # 维度4: 内容语义检查(调用本地大模型)
        if self._needs_semantic_check(block):
            llm_score = self._llm_validate(block['content'])
            score *= llm_score
        
        return score
```

### 2.2 数字标准化引擎

#### 2.2.1 技术难点
审计场景中数字格式多样:
- 千分位: `1,234,567`
- 中文单位: `123.4万`、`1.2亿`
- 小数点: `1234567.89`
- 负数表示: `(1234567)` 或 `-1234567`

需要统一标准化以支持准确比对。

#### 2.2.2 解决方案

```python
# number_normalizer.py
class NumberNormalizer:
    def normalize(self, text):
        """统一数字格式标准化"""
        
        # 处理千分位
        text = text.replace(',', '')
        
        # 处理中文单位
        if '万' in text:
            num = float(text.replace('万', ''))
            return num * 10000
        elif '亿' in text:
            num = float(text.replace('亿', ''))
            return num * 100000000
        
        # 处理括号表示负数
        if text.startswith('(') and text.endswith(')'):
            return -float(text[1:-1])
        
        # 处理小数点
        return float(text)
    
    def compare_numbers(self, num_a, num_b, tolerance=0.01):
        """数字比对(支持容差)"""
        normalized_a = self.normalize(num_a)
        normalized_b = self.normalize(num_b)
        
        diff = abs(normalized_a - normalized_b)
        return diff <= tolerance
```

### 2.3 跨页表格合并

#### 2.3.1 技术难点
- 表格跨页时需要识别表头与数据行
- 单元格坐标需要重新计算
- 合并单元格信息需要保留

#### 2.3.2 解决方案

OpenDataLoader已提供`previous_table_id`和`next_table_id`字段,基于此实现合并逻辑:

```python
# table_merger.py
class TableMerger:
    def merge_cross_page_tables(self, pages):
        """检测并合并跨页表格"""
        
        merged_tables = []
        
        for page in pages:
            for table in page['tables']:
                # 检查是否有跨页标记
                if table.get('next_table_id'):
                    # 查找下一页的表格
                    next_table = self._find_table_by_id(
                        pages, 
                        table['next_table_id']
                    )
                    
                    if next_table:
                        # 合并表格
                        merged = self._merge_tables(table, next_table)
                        merged_tables.append(merged)
                else:
                    merged_tables.append(table)
        
        return merged_tables
    
    def _merge_tables(self, table_a, table_b):
        """合并两个表格"""
        # 保留第一个表格的表头
        header = table_a['rows'][0]
        
        # 合并数据行
        data_rows = table_a['rows'][1:] + table_b['rows'][1:]
        
        # 重新计算行索引
        for idx, row in enumerate(data_rows):
            for cell in row['cells']:
                cell['row'] = idx + 1
        
        return {
            'rows': [header] + data_rows,
            'bbox': self._calculate_merged_bbox(table_a, table_b),
            'page_range': [table_a['page'], table_b['page']]
        }
```

### 2.4 分屏比对界面实现

#### 2.4.1 技术难点

**难点1: 坐标系转换**

OpenDataLoader输出的坐标是PDF内部坐标系(单位pt,原点在左下角),而PDF.js渲染到Canvas时有缩放和坐标翻转。

```
PDF坐标系              PDF.js Canvas坐标系
原点: 左下角           原点: 左上角
单位: pt (1/72 inch)   单位: px
受缩放影响             受zoom level + devicePixelRatio影响
```

**难点2: 双侧高度不一致的同步滚动**

左侧PDF是固定页面高度,右侧结构化内容是自适应高度,滚动位置无法1:1对应。

**难点3: 图片和图表内容渲染**

- 嵌入图片: 可通过base64直接显示
- 图表: 只能显示AI描述,无法重现原始图表
- 印章: 提取文字+标记需人工核验

#### 2.4.2 解决方案

**坐标转换实现**:
```javascript
// CoordinateMapper.js
class CoordinateMapper {
    constructor(pdfPage, scale) {
        this.pageHeight = pdfPage.view[3]; // PDF页面高度
        this.scale = scale;
    }
    
    // PDF坐标 → Canvas坐标
    pdfToCanvas(bbox) {
        const [x1, y1, x2, y2] = bbox;
        return {
            left: x1 * this.scale,
            top: (this.pageHeight - y2) * this.scale, // Y轴翻转
            width: (x2 - x1) * this.scale,
            height: (y2 - y1) * this.scale
        };
    }
    
    // Canvas坐标 → PDF坐标(用于点击反查)
    canvasToPdf(x, y) {
        return {
            x: x / this.scale,
            y: this.pageHeight - (y / this.scale)
        };
    }
    
    // 更新缩放比例时重算所有高亮框
    updateScale(newScale) {
        this.scale = newScale;
        // 触发重新渲染所有高亮框
    }
}
```

```javascript
// ScrollSyncer.js
class ScrollSyncer {
    syncScroll(sourcePanel, targetPanel, blockMapping) {
        // 找到源面板视口中心的元素
        const sourceRect = sourcePanel.getBoundingClientRect();
        const centerY = sourceRect.top + sourceRect.height / 2;
        
        // 遍历所有block,找到最接近中心的
        let closestBlock = null;
        let minDistance = Infinity;
        
        for (const block of sourcePanel.querySelectorAll('[data-block-id]')) {
            const blockRect = block.getBoundingClientRect();
            const blockCenterY = blockRect.top + blockRect.height / 2;
            const distance = Math.abs(blockCenterY - centerY);
            
            if (distance < minDistance) {
                minDistance = distance;
                closestBlock = block;
            }
        }
        
        // 滚动目标面板到对应block
        if (closestBlock) {
            const blockId = closestBlock.dataset.blockId;
            const targetBlock = targetPanel.querySelector(
                `[data-block-id="${blockId}"]`
            );
            
            if (targetBlock) {
                targetBlock.scrollIntoView({
                    block: 'center',
                    behavior: 'smooth'
                });
            }
        }
    }
}
```

### 2.5 跨文件比对引擎

#### 2.5.1 技术难点

两份文档没有公共坐标系，只能通过内容语义建立映射：

```
文件A 第5页 "货币资金 1,234,567"
 └── bbox: [120, 100, 400, 120] ← A文件坐标系

文件B 第2页 "货币资金 1,284,567"
 └── bbox: [95, 340, 380, 360]  ← B文件坐标系，完全不同
```

#### 2.5.2 解决方案

**页面对齐（调用本地大模型）**:
```python
# 提取每页语义摘要，用大模型计算相似度，自动推荐映射关系
page_a_summary = extract_page_summary(doc_a, page=5)
page_b_summary = extract_page_summary(doc_b, page=2)

similarity = llm.compare(page_a_summary, page_b_summary)
if similarity > 0.8:
    suggest_mapping(page_a=5, page_b=2)
```

**元素级语义匹配**:
```python
# semantic_matcher.py
class SemanticMatcher:
    def match_elements(self, doc_a_blocks, doc_b_blocks):
        matches = []

        for block_a in doc_a_blocks:
            best_match = None
            best_score = 0

            for block_b in doc_b_blocks:
                # 类型必须一致
                if block_a['type'] != block_b['type']:
                    continue

                # 多维度相似度: 文本相似度(0.7权重) + 相对位置(0.3权重)
                text_sim = self._text_similarity(block_a['content'], block_b['content'])
                pos_sim  = self._position_similarity(block_a['bbox'], block_b['bbox'])
                score    = 0.7 * text_sim + 0.3 * pos_sim

                if score > best_score and score > 0.75:
                    best_score = score
                    best_match = block_b

            if best_match:
                matches.append({
                    'doc_a':      block_a,
                    'doc_b':      best_match,
                    'similarity': best_score,
                    'diff_type':  self._get_diff_type(block_a, best_match)
                })

        return matches
```

#### PDFViewer.vue — 坐标转换与高亮层

```vue
<template>
    <div class="pdf-viewer" ref="container">
        <div class="controls">
            <el-button-group>
                <el-button @click="zoomOut">-</el-button>
                <el-button>{{ Math.round(scale * 100) }}%</el-button>
                <el-button @click="zoomIn">+</el-button>

        <div class="pdf-canvas-wrapper" @scroll="handleScroll">
            <canvas
                v-for="page in pages"
                :key="page.num"
                :ref="el => setCanvasRef(page.num, el)"
            />

            <!-- SVG高亮层叠加在Canvas上 -->
            <svg class="highlight-overlay">
                <rect
                    v-for="h in visibleHighlights"
                    :key="h.id"
                    :x="h.x" :y="h.y"
                    :width="h.width" :height="h.height"

<script setup>
import { ref, computed, onMounted, watch } from 'vue'
import * as pdfjsLib from 'pdfjs-dist'

const props = defineProps({ pdfUrl: String, highlights: Array, scale: Number })
const emit  = defineEmits(['scale-change', 'click-block', 'scroll'])

const pages     = ref([])
const canvasRefs = new Map()

onMounted(async () => {
    const pdf = await pdfjsLib.getDocument(props.pdfUrl).promise
    for (let i = 1; i <= pdf.numPages; i++) {
        const page = await pdf.getPage(i)
        pages.value.push({ num: i, page })
        renderPage(page, i)
    }
})

const renderPage = async (page, pageNum) => {
    const canvas   = canvasRefs.get(pageNum)
    const viewport = page.getViewport({ scale: props.scale })
    const context  = canvas.getContext('2d')
    canvas.width   = viewport.width
    canvas.height  = viewport.height
    await page.render({ canvasContext: context, viewport }).promise
}

// 缩放变化时重新渲染所有页面
watch(() => props.scale, () => {
    pages.value.forEach(({ page, num }) => renderPage(page, num))
})

// Y轴翻转坐标转换（PDF原点左下 → Canvas原点左上）
const visibleHighlights = computed(() =>
    props.highlights.map(h => {
        const [x1, y1, x2, y2] = h.bbox
        return {
            ...h,
            x: x1 * props.scale,
            y: (h.pageHeight - y2) * props.scale,
            width:  (x2 - x1) * props.scale,
            height: (y2 - y1) * props.scale
        }
    })
)

// 滚动同步：以视口中心元素为锚点
const handleScroll = () => {
    const centerBlockId = findBlockAtViewportCenter()
    emit('scroll', centerBlockId)
}

const zoomIn  = () => emit('scale-change', props.scale * 1.2)
const zoomOut = () => emit('scale-change', props.scale / 1.2)
const setCanvasRef = (pageNum, el) => { if (el) canvasRefs.set(pageNum, el) }
</script>
```

```javascript
// stores/compare.js
import { defineStore } from 'pinia'
import { ref, computed }  from 'vue'

export const useCompareStore = defineStore('compare', () => {
    const documentA    = ref(null)
    const documentB    = ref(null)
    const diffs        = ref([])
    const activeBlockId = ref(null)

    const reviewedDiffs = computed(() => diffs.value.filter(d => d.reviewed))

    const loadDocuments = async (docAId, docBId) => {
        const [resA, resB] = await Promise.all([
            fetch(`/api/documents/${docAId}`),
            fetch(`/api/documents/${docBId}`)
        ])
        documentA.value = await resA.json()
        documentB.value = await resB.json()
    }

    const loadDiffs = async (docAId, docBId) => {
        const res     = await fetch(`/api/compare/${docAId}/${docBId}`)
        diffs.value   = await res.json()
    }

    const setActiveBlock = (blockId) => { activeBlockId.value = blockId }

    const markReviewed = async (diffId) => {
        const diff = diffs.value.find(d => d.id === diffId)
        if (diff) {
            diff.reviewed = true
            await fetch(`/api/diffs/${diffId}/review`, { method: 'POST' })
        }
    }

    return { documentA, documentB, diffs, activeBlockId, reviewedDiffs,
             loadDocuments, loadDiffs, setActiveBlock, markReviewed }
})
```

| 大文件处理性能 | 500+页处理可能极慢 | 分块处理（每次50页）+ Celery异步 + WebSocket实时进度推送 |
| OpenDataLoader无置信度输出 | 无法直接标记低置信度区域 | 自研启发式置信度算法（字体异常/bbox重叠/OCR层检测） |
| 跨文件坐标系不兼容 | 无公共坐标，无法直接映射 | 通过语义相似度建立内容映射，分别在各自文档中独立高亮 |

## 八、架构关键设计原则

1. **不修改OpenDataLoader源码**: 采用Wrapper模式，保持可升级性，降低耦合
2. **坐标一致性**: 右侧所有元素（包括图片）均基于bbox还原尺寸，与左侧保持视觉一致
3. **锚点同步策略**: 滚动同步以内容元素为锚点，而非滚动百分比，解决左右高度不一致问题
4. **渐进式加载**: 大文件分块处理，WebSocket推送进度，避免长时间白屏
5. **审计优先**: 所有低置信度内容强制标记need_review，宁可多审也不漏审
